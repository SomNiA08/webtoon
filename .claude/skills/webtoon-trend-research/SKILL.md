# 웹툰 트렌드 리서치 방법론

리서치팀(trend-scout, platform-ranker, audience-analyst, hook-analyst, trend-synthesizer)이 "지금 무엇을, 어떤 톤·구조로 만들면 인기 있을지"를 데이터로 답하기 위한 공통 방법론이다. 명령형으로 따른다.

조사 대상이 빠르게 변하므로, 결론보다 **근거와 출처**를 먼저 챙긴다. 모든 주장에는 출처와 관측 일자를 붙인다. 상충 정보는 양쪽을 적고 출처를 병기한다(임의로 한쪽을 지우지 않는다).

## 왜 이렇게 조사하는가 (Why-first)

- 웹툰 흥행은 우연이 아니라 **반복되는 패턴**이다. 패턴을 찾아 이식하면 흥행 확률이 오른다. 그래서 한 작품이 아니라 "여러 작품에서 반복되는가"를 본다.
- 트렌드는 **시점 의존적**이다. 6개월 전 핫했던 트로프가 지금은 식었을 수 있다. 그래서 관측 일자를 항상 기록하고 최근 신호를 우선한다.
- 기획팀은 **실행 가능한 결정**을 원한다. "로판이 인기"가 아니라 "로판 × 회귀 × 차가운 복수극 톤 × 1화 끝 정체 반전"처럼 곧바로 쓸 수 있는 형태로 정리해야 가치가 있다.
- 시나리오/비주얼 팀이 산출물을 소비하므로, 발견은 **이야기로 전환 가능한 형태**(트로프·기법·구조)로 남긴다.

## 무엇을 어디서 조사하는가

네 축으로 조사한다. 자신의 역할 축을 깊게 파되, 다른 축의 신호도 교차 확인한다.

### 1) 장르·트로프 (trend-scout)
- 인기 장르: 회귀·빙의·환생(회빙환), 다크판타지, 로맨스판타지(로판), 헌터/게이트물, 사이다 서사, 먼치킨, 학원/직장 복수극, 아카데미물, 현대판타지, 무협, 일상/힐링 등.
- 각 장르의 반복 트로프와 변주, 그리고 라이프사이클(상승 / 정점 / 하락)을 구분한다.
- 장르 혼종(로판+회귀, 헌터물+복수극 등)과 신규 결합의 초기 신호를 잡는다.
- 트로프 뒤의 **독자 욕구**(대리만족·통제감·공정함·설렘)를 한 줄로 해석한다.

### 2) 플랫폼·연재 구조 (platform-ranker)
- 플랫폼: 네이버웹툰, 카카오페이지, 카카오웹툰, 레진코믹스(필요 시 리디·투믹스 등).
- 랭킹을 종류별로 본다: 인기 / 유료 / 신작 / 완결. 플랫폼마다 랭킹 산정 방식이 다름을 전제한다.
- 인기작의 **연재 구조**: 회차당 분량(컷·패널 수, 스크롤 길이), 회차 호흡, 에피소드 단위.
- **클리프행어 위치**: 회차 끝 컷의 긴장 처리, "다음화"를 누르게 하는 지점.
- **결제 전환 구간**: 무료→유료 분기 회차, 기다무/미리보기 컷오프, 유료 구간의 훅 밀도.

### 3) 독자·반응 (audience-analyst)
- 인구통계: 장르별 주 독자층(연령·성별·플랫폼 성향). 대부분 추정이므로 근거를 적고 "추정"으로 라벨링한다.
- **몰입 요소**: 붙잡는 감정 동인(대리만족·사이다·설렘·통제감·정의 실현)과 그 트리거 장면.
- **이탈(드롭) 지점**: 떠나는 전형 구간(느린 도입부, 고구마 누적, 떡밥 방치, 전개 지연, 작화 붕괴, 결제 장벽)을 회차·전개 단계와 연결한다.
- **댓글 반응 패턴**: 베스트 댓글을 유형으로 코딩(사이다 환호 / 고구마 불만 / 떡밥 추리 / 캐릭터 호불호 / 결제 불만). 개별 폭언이 아닌 반복 패턴으로 집계한다.

### 4) 후킹·클리프행어·반전 (hook-analyst)
- **1화 후킹 공식**: 첫 컷~첫 회차의 낚는 장치(강렬한 의문, 즉시 위기, 비범한 주인공 선언, 충격 오프닝, 회귀/각성 트리거)와 배치 순서.
- **클리프행어 메커니즘**: 회차 끝 긴장을 끊는 위치와 방식.
- **반전(트위스트) 역설계**: 어떤 예측을 어떻게 뒤집는가(정체·배신·관계 역전·시점·진실 공개)와 충격 극대화 타이밍.
- **복선 기법**: "심기 → 숨기기(미스디렉션) → 터뜨리기"의 단서와 회수 간격. 공정한 반전은 앞에 단서가 있다.

역설계의 관찰 프레임(반전·클리프행어 분해 체크리스트)은 길어서 분리해 두었다. `references/observation-frames.md`를 참고한다.

## 신뢰할 출처 우선순위

상위일수록 신뢰한다. 가능하면 1·2차 출처를 우선하고, 단정 전에 교차 확인한다.

1. **플랫폼 공식 데이터** — 각 플랫폼의 랭킹/인기/신작 페이지, 공식 발표·IR·보도자료(조회수·매출·랭킹 산정).
2. **플랫폼 내 1차 신호** — 작품 회차 구조, 평점, 댓글/베스트 댓글, 무료↔유료 분기 표시.
3. **업계 보도·집계** — 웹툰 전문 매체, 산업 리포트, 신뢰도 있는 차트 집계.
4. **커뮤니티·리뷰·해설** — 독자 커뮤니티, 리뷰 블로그, 작품 해설. 표본 편향을 전제로 "패턴"만 취한다.
5. **2차 요약·추정** — 위 접근이 막혔을 때만. 반드시 "추정/2차 출처"로 라벨링한다.

규칙:
- 모든 항목에 **URL과 관측 일자**를 붙인다. 랭킹은 스냅샷이므로 시점이 빠지면 무효다.
- **상충 정보는 양쪽 병기**한다. 예: "출처 A는 로판 상승, 출처 B는 정체 — 양쪽 출처 명시." 한쪽을 지우지 않는다.
- 접근 불가 출처는 URL과 실패 사유를 남겨 후속 재시도가 가능하게 한다.
- 검증 안 된 효과·수치는 "추정" 또는 "가설"로 라벨링한다. 추측을 사실처럼 적지 않는다.

## 도구 사용

- `WebSearch` — 트렌드·랭킹·작품·반응을 탐색한다. 결과가 빈약하면 검색어를 한국어 / 영어 / 플랫폼명 / 작품명으로 바꿔 재시도한다.
- `WebFetch` — 랭킹 페이지·기사·해설 본문을 가져와 구조·수치를 추출한다. 동적 로딩·로그인으로 막히면 2차 출처로 보완하고 한계를 명시한다.

## 종합 브리프의 구조 (trend-synthesizer)

네 보고서를 **교차 검증**해 실행 가능한 기획 인사이트로 수렴시킨다. 평균이 아니라 판단이다. 모든 권장에 근거(어느 보고서의 어떤 신호)를 붙인다. `trend-brief.md`는 다음을 담는다.

1. **한 줄 결론** — "지금은 [장르×트로프]를 [톤]으로, [회차 구조]로 만들라."
2. **권장 장르·트로프 조합** — 1순위 + 대안, 각 근거.
3. **권장 톤·분위기** — 감정 타깃과 이유.
4. **권장 회차 구조** — 50+ 패널 전제, 클리프행어 배치, 매 회차 반전 지점, 결제 친화 구조.
5. **차별점(엣지)** — 식상 트로프를 비트는 지점, 신규 결합.
6. **후킹·반전 전략** — 채택한 패턴 카드와 적용 위치.
7. **타깃 독자·이탈 회피** — 핵심 독자와 드롭 방지 설계.
8. **리스크/확인 필요** — 약한 신호, 데이터 공백, 검증 필요 가정.
9. **근거 출처 요약** — 결론별 출처 트레이스.

종합은 항상 콘텐츠 품질 핵심을 권장안에 녹인다: **대사 위주 · 고긴장 유지 · 매 회차 반전 1개 이상 · 회차당 50+ 패널 · 캐릭터/세계관/톤 일관성.** 이 기준으로 후속 팀이 자연히 향하게 만든다.

## 출력 파일 경로 (반드시 준수)

작업 루트는 `_workspace/`. 자신의 역할에 해당하는 파일에만 쓴다.

- trend-scout → `_workspace/01_research/trend-scout.md`
- platform-ranker → `_workspace/01_research/platform-ranker.md`
- audience-analyst → `_workspace/01_research/audience-analyst.md`
- hook-analyst → `_workspace/01_research/hook-analyst.md`
- trend-synthesizer → `_workspace/01_research/trend-brief.md` (위 4개를 Read하여 종합)
- 공통 입력: `_workspace/00_input/brief.md` (사용자 입력·회차 번호·제약)

각 보고서는 자기 마크다운 섹션 구조(요약 → 본문 → 기획 시사점 → 출처)를 갖추고, 끝에 출처를 모은다. 종합본의 직접 소비자는 시나리오팀의 concept-architect · series-plotter · twist-master이므로, 발견은 회차 단위로 이식 가능한 형태로 남긴다.

## 후속 작업 (다시 / 재조사 / 업데이트 / 보완)

- 해당 출력 파일이 이미 있으면 **Read하여 변경분만 갱신**한다. 전체를 재작성하지 않는다.
- 랭킹·트렌드는 관측 일자를 갱신하고, 라이프사이클 단계(상승·정점·하락)를 업데이트한다.
- 사용자 피드백("이 장르 더 파봐", "최신으로 갱신", "리스크 보수적으로")이 있으면 해당 섹션만 보강한다.
- 종합본은 4개 보고서가 갱신되면 변경된 결론과 근거 트레이스를 함께 업데이트한다.
